## Improve first

## Design decision

Turn the model-slicing form into three compact horizontal bands inside the existing outer `.panel`:

1. **Input and layering band** — a single full-width section with STL file, fiber JSON, layer height, build axis, the existing fiber note, and the existing collapsed Z/tolerance details.
2. **Path kernel band** — a single full-width 12-column section that packs the default Prusa controls into complete rows with no orphaned field.
3. **Process and action band** — raft, curved-Z, submit button, and status share one final row.

Use the existing spacing, border, radius, color, and control-height tokens. Do not shrink text below the current 13px label baseline or controls below the current 36px minimum height. The reduction must come from composition and removal of empty grid cells, not from making controls hard to read or operate.

The live geometry that governs this plan is fixed and must be preserved unless a later measurement proves it changed:

- Browser viewport: `1080 × 1695px`.
- Main container: `960px` wide, with `20px` horizontal padding.
- Model-slicing panel: `920px` wide, with `20px` padding.
- Form content: `878px` wide.
- Current form columns: two `433px` tracks with a `12px` gap.
- Result/preview width: `920px`.

For that measured `878px` form width:

- Use `grid-template-columns: repeat(12, minmax(0, 1fr))` with the existing `--space-2` (`8px`) gap for band internals. This yields approximately `65.8px` per base track before spans, so a 4-track field is about `287px`, a 3-track field about `213px`, and a 2-track field about `140px`.
- Input band: STL file `span 4`, fiber JSON `span 4`, layer height `span 2`, build axis `span 2`; fiber note `span 8`; advanced layer details `span 4` and remain collapsed by default.
- Kernel band row 1: nominal width, measured width, perimeter count, and slicing kernel each `span 3`; keep “打印内外轮廓” inside the perimeter field group.
- Kernel band row 2: infill pattern `span 4`; the two optimization toggles share `span 8` in a compact inline/wrapping option group.
- Kernel band row 3: fill density, line overlap, and contour overlap each `span 4`.
- Kernel band row 4: advanced smoothing details span all 12 columns and remain collapsed. The hidden PySLM native settings also span all 12 columns when activated; preserve its current internal groups and visibility logic.
- Process band: raft group `span 4`, curved-Z group `span 5`, action/status group `span 3`. Align the action group to the end of the band without a fixed minimum height.
- At the existing `820px` breakpoint, use six columns and allow semantically paired controls to wrap together; at `520px`, retain the existing single-column fallback.

The accepted default Prusa state must reduce the `878px`-wide form from the measured `1114.98px` to at most `580px`, keep the complete model-slicing panel at or below `665px`, and move the preview top from `y=1664.81px` to `y≤1150px` in the live `1080 × 1695` viewport. At least `545px` of the preview should then be visible without scrolling.
